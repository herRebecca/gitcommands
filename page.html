GIT COMMANDS

GIT INIT

The git init command is a fundamental Git operation used to start a new repository. It's the first command you run to initialize a new Git project or to reinitialize an existing one.

If no directory is specified, git init initializes the **current** directory as a Git repository. Always remember to specify the directory if you want to initialize elsewhere.

It comes with some options to customize the initialization process:

  • --bare: Initializes the repository as a bare repo (no working directory). Best for server-side use.
  • --shared[<permissions>]: Sets up the repo for collaboration between multiple users with specific permissions.

Use git init when:

  • Starting a new project from scratch
  • Adding version control to an existing project

✅ Safe to use multiple times: Running git init again on the same directory won’t overwrite existing data in the Git directory.

GIT ADD

The git add command is used to stage changes in your working directory for the next commit. It allows you to select which changes you want to include in the next commit, making it a crucial step in the Git workflow.

When you run git add, you can specify individual files, directories, or use wildcards to stage multiple files at once. For example:

  • git add file.txt stages a specific file.
  • git add . stages all changes in the current directory and its subdirectories.
  • git add *.txt stages all text files in the current directory.

You can also use git add with options like -A to stage all changes (including deletions) or -u to stage only tracked files that have been modified or deleted.

Remember that git add does not commit the changes; it only stages them. You need to follow up with git commit to create a new commit with the staged changes.

GIT COMMIT

The git commit command is used to save changes to your local repository. It creates a new commit with the staged changes, allowing you to track the history of your project.

When you run git commit, you can include a message that describes the changes made in that commit. This message is important for understanding the purpose of the commit later on.

For example:

  • git commit -m "Added new feature" creates a commit with the specified message.
  • git commit -a stages all modified files and commits them in one step.

Remember that git commit only affects your local repository. To share your changes with others, you need to push them to a remote repository using git push.

GIT BRANCH

The git branch command is used to manage branches in your Git repository. Branches allow you to work on different features or versions of your project simultaneously without affecting the main codebase.

When you run git branch, it lists all the branches in your repository and highlights the current branch you are on. You can also create new branches, delete existing ones, and rename them.

For example:

  • git branch new-feature creates a new branch called "new-feature".
  • git branch -d old-feature deletes the "old-feature" branch.
  • git checkout -b another-feature creates and switches to a new branch called "another-feature".

Branches are essential for collaborative development, allowing multiple developers to work on different features without interfering with each other's work. You can merge branches later using git merge.
