GIT COMMANDS

GIT INIT

The git init command is a fundamental Git operation used to start a new repository. It's the first command you run to initialize a new Git project or to reinitialize an existing one.

If no directory is specified, git init initializes the **current** directory as a Git repository. Always remember to specify the directory if you want to initialize elsewhere.

It comes with some options to customize the initialization process:

  • --bare: Initializes the repository as a bare repo (no working directory). Best for server-side use.
  • --shared[<permissions>]: Sets up the repo for collaboration between multiple users with specific permissions.

Use git init when:

  • Starting a new project from scratch
  • Adding version control to an existing project

✅ Safe to use multiple times: Running git init again on the same directory won’t overwrite existing data in the Git directory.
