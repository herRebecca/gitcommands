# 📘 Git Practice Repository

Welcome to the Esther's Practice Repository — a collaborative project designed to help me learn and practice Git through real-world workflows.

Each contributor would create a branch focused on a specific Git command.  (I contributed to most of it anyways )
This main branch serves as the base, containing the homepage, table of contents,setup instructions and of course this README file.

---

## 🎯 Project Purpose

- Practice Git branching and collaboration.
- Document individual Git commands in isolated branches.
- Learn by doing — hands-on with commits, branches, merges, and pull requests.

---

## 🏗️ How This Works

1. **Clone this repository** from the `main` branch:
   
   git clone https://github.com/herRebecca/git-practice-repo.git
   cd git-practice-repo
Create a new branch named after a Git command you want to practice:

git checkout -b git-status
Add content to your branch that explains or demonstrates the command with html and css

No need to edit the main branch files. (Id lock it anyways)


2. **Push your branch to GitHub**:

git push "origin" git-status
   **for examplee**

🤝 Contributing
Collaborators added would be given access to contribute 
Just follow these simple rules:

Use a clear and consistent branch name (e.g., git-fetch, git-rebase).

Only commit to your branch!!

Add meaningful content that explains or demonstrates the Git command.

You do not need to open a pull request unless you'd like a review or want your branch merged into another.
